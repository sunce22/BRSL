# Ukrainian Localization (i18n) — Design Spec

**Date:** 2026-05-30
**Status:** Approved

---

## Goal

Add English/Ukrainian language toggle to Panel and OBS overlays. Hero names, skill names, skill descriptions, effect descriptions, and all UI strings switch language on demand. Translations stored in separate i18n files; heroes.json unchanged.

## Architecture

Three layers:

1. **i18n data files** — static JSON, loaded once at startup
2. **`shared/i18n.js`** — loads, caches, exposes `t(key)` and `translateHero(hero)`
3. **UI entry points** — `panel.js`, `obs.js`, `obs-guest.js` — call `translateHero()` before render, `t()` for UI strings; re-render on lang switch

## `shared/i18n.js`

```javascript
const LANG_KEY = 'rsl-lang';

export function getLang() {
  return localStorage.getItem(LANG_KEY) ?? 'en';
}

export function setLang(lang) {
  localStorage.setItem(LANG_KEY, lang);
}

let _heroTr = null;   // heroes-uk.json cache
let _uiTr   = null;   // ui-uk.json cache

// Lazy: called only when user switches to 🇺🇦. Safe fallback if files missing.
export async function loadTranslations(basePath = '..') {
  const [heroesResult, uiResult] = await Promise.allSettled([
    fetch(`${basePath}/i18n/heroes-uk.json`).then(r => r.json()),
    fetch(`${basePath}/i18n/ui-uk.json`).then(r => r.json()),
  ]);
  _heroTr = heroesResult.status === 'fulfilled' ? heroesResult.value : {};
  _uiTr   = uiResult.status   === 'fulfilled' ? uiResult.value   : {};
}

// Called from panel.js/obs.js only when lang === 'uk' is selected
// English startup does NOT call loadTranslations → no extra HTTP requests

// UI string lookup — EN strings are hardcoded as fallback
const UI_EN = {
  tab_heroes: 'Heroes', tab_effects: 'Effects', tab_support: '♥',
  search_heroes: 'Search heroes...', search_effects: 'Search effects...',
  back: '← Back', cooldown_prefix: 'CD:', timer_label: 'Timer',
  pin_label: 'Pin', dismiss_label: 'Dismiss',
  feedback_placeholder: 'Your feedback...', feedback_submit: 'Send',
  feedback_sending: 'Sending...', feedback_ok: 'Sent!',
  feedback_error: 'Error. Try again.', support_title: 'Support',
  error_loading: 'Error loading data:',
};

export function t(key) {
  if (getLang() === 'uk' && _uiTr?.[key]) return _uiTr[key];
  return UI_EN[key] ?? key;
}

// Returns hero object with translated fields (or original if no translation)
export function translateHero(hero) {
  if (getLang() === 'en') return hero;
  const tr = _heroTr?.[hero.id];
  if (!tr) return hero;
  return {
    ...hero,
    name: tr.name ?? hero.name,
    skills: hero.skills.map((s, i) => ({
      ...s,
      name:        tr.skills?.[i]?.name        ?? s.name,
      description: tr.skills?.[i]?.description ?? s.description,
    })),
  };
}
```

## UI Language Toggle

### Placement

**OBS** (top-right, before ☰ button):
```
[🇬🇧] [🇺🇦]  [☰]
```

**Panel** (tab bar, after ♥ support button):
```
[Heroes] [Effects] [♥]  [🇬🇧][🇺🇦]
```

### Behavior
- Active lang: opacity 1.0, cursor default
- Inactive lang: opacity 0.4, cursor pointer, hover → opacity 0.7
- On click: `setLang(lang)` → re-render entire current view (same function as tab switch)
- Persisted in `localStorage` key `rsl-lang`

### HTML (shared pattern)
```html
<div class="lang-toggle">
  <button class="lang-btn" data-lang="en" title="English">🇬🇧</button>
  <button class="lang-btn" data-lang="uk" title="Українська">🇺🇦</button>
</div>
```

### CSS
```css
.lang-toggle { display: flex; gap: 2px; align-items: center; }
.lang-btn {
  background: none; border: none; cursor: pointer;
  font-size: 16px; padding: 2px 4px; opacity: 0.4;
  border-radius: var(--radius); transition: opacity 0.15s;
}
.lang-btn:hover { opacity: 0.7; }
.lang-btn--active { opacity: 1.0; cursor: default; }
```

## Rendering Changes

All entry points call `translateHero(hero)` before passing to `renderHeroCard()` or rendering hero list items. Effect descriptions lookup via `t()` or extended effect-descriptions map.

Files modified:
| File | Change |
|------|--------|
| `panel.js` | import `{t, translateHero, loadTranslations, getLang, setLang}`; call `loadTranslations()` in `init()`; wrap hero renders with `translateHero()`; replace hardcoded strings with `t()`; add lang-toggle to tab bar |
| `obs.js` | same imports; add lang-toggle next to ☰; re-render card on lang switch |
| `obs-guest.js` | same as obs.js |
| `hero-card.js` | receives already-translated hero — no change needed |
| `effects-list.js` | effect labels via `t()` or effect slug map extended to uk |
| `effect-descriptions.js` | `highlightDescription()` — effect names in descriptions stay EN (they're game terms) |
| `panel.css` / `obs.css` | add `.lang-toggle`, `.lang-btn` styles |
| `shared/styles.css` | shared lang-toggle styles |

## Translation Extraction Pipeline

Run separately, not part of UI build. Requires game running.

**`tools/extract-l10n-uk.py`**
- Opens Raid.exe process (ReadProcessMemory, same approach as `scan-hero-memory.py`)
- Searches for Ukrainian UTF-16 LE strings near known skill_id integers
- Output: `tools/l10n-uk-raw.json`

```json
{
  "5780_s1": { "name": "Пекельний вузол", "desc": "Атакує 1 ворога..." },
  "5780_s2": { "name": "Боже осяяння", "desc": "..." }
}
```

Key format: `{hero_type_id}_s{skill_index}`.

**Security — sanitize raw memory strings:**
All strings from `l10n-uk-raw.json` must be sanitized before writing `heroes-uk.json`:
```python
import html
def sanitize(s: str) -> str:
    return html.escape(s, quote=True)  # &, <, >, ", ' → entities
```
Apply to every `name` and `desc` field. Game memory is not trusted as safe HTML.

**`tools/build-heroes-uk.py`**
- Reads `tools/l10n-uk-raw.json` + `tools/hero-num-to-name-v2.json` + `data/heroes.json`
- Builds `i18n/heroes-uk.json`
- Reports coverage: N/407 heroes translated

## Scope Boundaries

**In scope:**
- Hero names, skill names, skill descriptions
- All UI strings in panel + OBS + OBS-guest
- Effect names in effects tab (slug labels)

**Out of scope (this iteration):**
- Effect descriptions body text (English game terms, keep EN)
- Faction names (keep EN — consistent with global RSL community)
- Rarity/affinity badge labels (keep EN)

---

## Security & Reliability Requirements

| # | Requirement |
|---|-------------|
| 1 | `loadTranslations()` uses `Promise.allSettled` — app works in EN even if i18n files missing/404 |
| 2 | `loadTranslations()` called **lazily** — only when user selects 🇺🇦 for first time, not on EN startup |
| 3 | `build-heroes-uk.py` runs `html.escape()` on every string from game memory before writing JSON |
| 4 | `highlightDescription()` in `effect-descriptions.js` already wraps output in safe HTML — verified unchanged |
| 5 | `escapeHtml()` called at render time on all `hero.name`, `skill.name` — covers both EN and UK strings |
